disable-model-invocation: true
allowed-tools: Read, Grep, Glob
---

목표:
- "무엇을 테스트할 것인가"를 행동(행위) 단위로 정의한다.
- 최소 5개 행동을 선정한다(테스트가 5개가 아니라, 검증할 행동이 5개).

절차:
1) API/CLI/배치 등 시스템 경계를 찾고(라우트 목록, 공개 함수, 주요 커맨드),
2) 사용자/외부 시스템이 실제로 기대하는 결과를 행동 문장으로 만든다.
    - 예: "주문 생성 시 주문 레코드 생성 + 재고 감소 + 결제 상태 반영"
3) 각 행동에 대해 아래 점수를 매긴 뒤 Top 5~8을 고른다.
    - 가치(사용 빈도/매출/핵심 플로우) 1~5
    - 리스크(실패 시 피해/변경 가능성/복잡도) 1~5
4) 선정 기준(왜 이 5개인가)을 명시한다.

출력 포맷:
- 행동 후보 리스트(최대 15): [행동문장] / 가치 / 리스크 / 근거(코드 위치 또는 API)
- 최종 선정 행동(최소 5): ...
- 선정 기준(요약): ...
