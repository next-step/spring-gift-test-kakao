# AI 활용 접근 방식

## 사용한 프롬프트 흐름

다음과 같은 단계적 질문을 통해 테스트 전략을 구체화하였다.

### 1단계: 핵심 기능 도출 및 코드 이해
> 이 프로젝트의 핵심 기능을 알려주고 어떤 사용자의 행위가 테스트할 가치가 있는지 알려줘.

### 2단계: 인수 테스트 관점 확장
> 인수테스트를 진행하는 관점에서 어떤 테스트를 진행해볼 수 있지?

### 3단계: 테스트 전략 문서화 요청
> 인수 테스트 진행할건데 어떤 행위에 대해서 테스트를 해야 하는지,
> 그리고 우선순위는 어떻게 두고 작업을 진행하면 좋을지, 테스트 전략 문서를 작성해보고 싶어. 담겨야 할 대표적인 내용은 다음과 같아.
> 추가적으로 있으면 좋을 것 같은 것도 너가 반영해서 작성해줘.

포함 요청 항목:
- 검증할 행위 목록
- 테스트 데이터 전략
- 검증 전략
- 주요 의사결정

### 4단계: 현재 구조 기반 claude.md 생성 요청
> 요구사항과 지금 프로젝트 구조를 바탕으로 claude.md 파일을 작성해줘

### 5단계: 시나리오 우선순위 전체 도출
> 지금 인수테스트 할만한 시나리오를 우선순위 순서로 전부 다 작성해줘

### 6단계: 커버리지 검증
> 지금 사용자가 할 수 있는 행위들을 전부 다 작성한거지? 모든 경우를 다 커버할 수 있는 시나리오인거지? 아니라면 보완해서 다시 알려줘

### 7단계: Skill 추출
> 지금 테스트코드 작성한 방식을 계속 비슷하게 적용하고 싶어서 skill을 만들어서 활용하고 싶은데 한 번 만들어볼래?

### 8단계: 특정 테스트 파일 커버리지 검증
> giftacceptancetest 파일에서는 이 시나리오에 대해 모두 커버하고 있는거야? 아니라면 추가해볼래?
