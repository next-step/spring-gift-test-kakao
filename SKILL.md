이 스킬이 활성화되면 클로드는 다음의 **BDD 검수 원칙**에 따라 응답하세요:

1. **사용자 중심의 행위(Behavior)인가?**
    - 단순 API 스펙(CRUD)이 아닌, "사용자가 ~하기 위해 ~한다"는 비즈니스 흐름인지 체크하세요.
    - 예: '상품 등록 API 검증' (X) -> '관리자가 판매할 상품을 진열한다' (O)

2. **검증 전략의 'Outside-In' 원칙 확인:**
    - DB 직접 조회를 지양하고, 다른 API(관찰 가능한 행위)를 통해 상태 변화를 검증하는지 확인하세요.

**[문서 포함 필수 항목]**
1. **검증할 행위 목록**
    - 각 행위가 "누가 무엇을 한다" 형태인가?
    - 이 행위가 시스템의 핵심 가치 흐름과 연결되는가?
    - 단순 CRUD 나열이 아닌가?
    - 제외된 중요한 행위는 없는가?
2. **테스트 데이터 전략**
    - 테스트 간 격리를 어떻게 보장하는가?
    - API 준비 vs SQL 준비 중 왜 그 방식을 선택했는가?
    - 데이터 전략이 행위 검증 방식과 일관되는가?
3. **검증 전략**: Black-box 테스트 관점에서 '무엇을 관찰할 것인가' 정의.
4. **주요 의사결정**: "왜 이 방식인가?"에 대한 기술적/비즈니스적 타당성.
